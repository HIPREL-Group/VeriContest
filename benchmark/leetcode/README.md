# Structure

Use the example structure shown in `./lc7`, collect and write:

- `description.md`: the problem description, collect it in `.md` format

- `tags`: tags for this problem, including `difficulty`, `topics` and `acceptance rate`

- `code.rs`: the executable code for this problem

- `spec.rs`: the specifications for this problem

- `code_spec.rs`: executable code + specifications

- `verified.rs`: the final Verus-verified version, including all necessary components
